TOGBankClassic - Guild Bank Management for Classic Era

Are you tired of logging into multiple bank alts to find items? Does your guild rely on character banks instead of the guild vault? TOGBankClassic is the solution!

TOGBankClassic is a powerful, feature-rich addon that transforms how your guild manages inventory across multiple bank characters. View, search, and request items from all your guild bank alts without ever logging out—all in one convenient interface.

What's New

v1.1.1 (2026-05-29) - Sorting Fixes & Random-Suffix Requests

This release fixes two inventory sorting problems and a mix-up where requests for items like "of the Tiger" vs "of the Monkey" could point at the wrong version.

  • "By Type" now keeps each material together and orders it by level. Sorting your inventory by type used to scatter the same kind of armor across the list (a run of cloth, then some leather, then more cloth). Now all cloth is grouped together, then all leather, then all mail/plate — and within each material the pieces are ordered by required level from highest to lowest (e.g. all plate: 50, 49, 48…), so each group reads in a clean, predictable order.
  • "Level" sorting now uses the item's required level. The Level (High to Low / Low to High) sort was ordering by an internal item-level value, which made the list look like it counted down and then started over. It now sorts by the level you need to use the item, so the order reads cleanly from high to low. The "Minimum/Maximum Required Level" search filters were affected by the same mix-up and are fixed too.
  • "By Type" now works in the Search window. Choosing "By Type" in the Search window previously did nothing; it now groups results the same way the main inventory does.
  • REQ-003: Requests for random-suffix gear show the right version. Items such as "Spiked Club of the Tiger" and "Spiked Club of the Monkey" are different items that share the same base. Before, a request for one could display the other's tooltip and be treated as fulfillable by the wrong one. Requests now remember exactly which version was asked for, so the tooltip and the bank/mail availability checks match the correct item. (Requests created before this update don't carry that detail and keep working as they did.)

v1.1.0 (2026-05-23) - Data Corruption Fix: Bogus Item Counts & Cleaner Commands

This release fixes a data-corruption bug that could cause individual gear items in the inventory view to show wildly inflated quantities (thousands of a weapon that physically can only exist once), and trims the slash-command list down to only the commands real players actually use.

  • Filter gear by level. The Search window has two new compact inputs labelled "Min lvl" and "Max lvl". Type a number into either (or both) to hide items outside that required-level range. Leave one empty for an open-ended range. Combines with the existing Type/Quality/Usable filters.
  • Search window filters are more compact. The filter dropdowns no longer take a full row each — they're now sized to their content and arranged in a row that automatically wraps to multiple lines based on how wide you've made the Search window. Resize the window wider to see more filters per row; narrower to stack them. Frees up vertical space for results.
  • Help icon and bottom-right pagination on the Search window. A new "?" help icon at the bottom-right explains how Search works. The "< Previous" / "Next >" full-width buttons are gone — they're now compact "<" and ">" icons next to the close button, freeing another row of space for results.
  • More tooltips throughout the Search window. Hover the "Min lvl", "Max lvl", "Usable", "Filter", or "Sort" labels for an explanation of what each does. The Sort tooltip no longer pops up while you're trying to click the dropdown.
  • One-click access to settings. A new gear icon now sits next to the help "?" icon at the bottom-right of the main guild bank window. Click it to open the addon's options panel directly — no more navigating through Escape → Options → AddOns → TOGBankClassic.
  • ITEM-004: Inflated weapon and armor counts are fixed. Gear items could show counts in the hundreds or thousands due to a bug where mail items were being double-counted into the bank view across sync cycles. The bug is fixed and a one-time cleanup runs automatically within 30 seconds of logging in. You'll see a chat message like "Ghost purge: removed X linkless gear ghost(s) from saved data" if any corrupt entries were found and cleaned up. Fresh data from your bankers will refill anything that gets removed.
  • ITEM-003: Gear items showing as duplicated rows in the inventory are fixed. Plain "Battlefell Sabre" and "Battlefell Sabre of Power" can no longer collide in a way that corrupts each other's counts.
  • Slash-command list cleaned up. Type /togbank help and you'll see a much shorter list — only the commands that are actually useful for players (open the UI, sync, share, debug logging, reset). Internal developer/debugging commands like deltastats, protocol, forcedelta, perfstats etc. are no longer shown to players. Officers still have wipeall, roster, compact, and the debuglog family for reporting bugs.
  • Behind the scenes: groundwork is laid for a future release that will dramatically reduce the network bandwidth used by guild bank syncs (5x or more smaller). No user-visible change in this release; the heavy lifting will come in v1.2 once the foundation is fully populated.

If you were previously seeing impossible numbers like "6237 Battlefell Sabre of Power" in your guild bank view, this release fixes the cause and cleans up the leftover bad data automatically. No action needed on your part.

v1.0.0 (2026-04-11) - First Stable Release: Fulfill Location Awareness & Polish

After months of active development and in-guild testing, we're shipping 1.0.0. This release rounds out the fulfillment workflow with location-aware button states and fixes the last rough edges found during real-world use.

  • Fulfill button location awareness: The fulfill button now shows distinct icons and contextual tooltips for items that can't be mailed immediately — wax letter icon when the item is in your mail inbox, paired bag+letter icons when it's split across bank and mail, and shortage icons (with exact counts) when you have some in bags but need to pick up more. No more guessing why the button isn't ready.
  • TOOLTIP-001: Item link "Bankers:" tooltip no longer shows stock data for characters who have left the guild — guild roster membership is now verified before displaying any banker's inventory
  • FULFILL-001: Fulfill button icon now correctly transitions away from the split (shovel) icon after a bag split completes — it was stuck because the bag-update event was calling the wrong draw function
  • HIGHLIGHT-001: Bagnon item highlighting now works for tradeskill recipes and patterns (Pattern:, Formula:, Plans:, Schematic:, Design:, Manual:) — the colon in these names was silently breaking Bagnon's search parser

v0.10.10 (2026-04-04) - Same-Name Item Variant Disambiguation

  • REQ-001: Same-name item variants (e.g. Punctured Voodoo Doll Priest/Warrior/Druid) are now correctly distinguished — requests store the numeric item ID and all fulfillment, highlighting, and mail detection use it for exact matching; old clients fall back to name-matching automatically
  • REQ-002: Hovering a request's item name now shows the tooltip for the correct class variant, not whichever one happened to appear first in guild inventory
  • ItemHighlight fix: Needed-quantity calculation now correctly subtracts already-fulfilled amounts (was always showing full quantity due to wrong field name)

v0.10.9 (2026-04-03) - Cancel Reason Overhaul, Fulfillment Sound Fix & Options Cleanup

  • Role-aware cancel reasons: Bankers see 6 flavour-text reasons when cancelling a member's request; members cancelling their own orders see 5 separate reasons — role detected automatically via IsBank()
  • MAIL-014: Order fulfillment sound now actually plays — the original PlaySound() call used a string name removed in Patch 7.3.0; fixed to use the correct numeric SoundKitID
  • Fulfillment sound toggle: New Options toggle "Play Sound on Order Fulfilled" (defaults on) for players who find the sound disruptive
  • Reset Database tooltip: The Reset Database button in Options now has a clear tooltip explaining what it does and that it cannot be undone
  • Removed dead Communication Protocol dropdown from Options — the setting was never read by any actual send/receive code

v0.10.8 (2026-04-02) - Requests UI Pagination & Copyable Item Text

  • Pagination: Requests window now shows 50 rows per page — no more game freezes when switching to "Any Banker" with 100+ requests
  • Copyable item names: Click any item name in the Requests window to highlight it, then Ctrl+C to copy
  • Item tooltips on hover: Mouse over any item name in Requests to see the full WoW item tooltip with stats and quality
  • "Requests synced." confirmation printed after a user-triggered /togbank sync completes

v0.10.7 (2026-04-02) - P2P Delivery Watchdog Tuning

  • P2P-032: Delivery watchdog raised from 60s to 180s — prevents premature sync failure during slow AceCommQueue drains (observed worst case: 68–70s for 9 KB payloads)

Core Features

Unified Inventory Management

  • One-Click Access: View the complete inventory of all guild bank characters in a single window
  • No Alt-Hopping: Never log out to check what's in stock—see everything at a glance
  • Real-Time Updates: Inventory automatically syncs across all guild members using the addon
  • Mail Integration: Track items in transit—see what's in the mailbox as part of total inventory
  • Item Tooltip: Hover any item anywhere in-game to see which bankers have it and their exact stock levels—no addon window needed

Powerful Search System

  • Guild-Wide Search: Search across all bank characters simultaneously—find any item instantly
  • Live Filtering: Results update as you type, showing exact quantities and locations
  • Sort Dropdowns: Visual dropdown menus with 6-7 sort modes (A-Z, By Type, By Rarity, By Level, By Bank, By Quantity, By Slot) and reverse order toggle
  • Banker Labels: See which banker character has each item in search results
  • Click to Request: Click any search result to open a request dialog with smart quantity suggestions
  • Availability Info: See max requestable amounts based on officer-configured limits

Smart Request System

  • In-Game Requests: Guild members request items directly through the addon—no whispers needed
  • Request Tracking: View all pending, fulfilled, and completed orders in the Requests window
  • Timeline Tooltips: Hover any request's date to see when it was submitted, filled, or cancelled (with reason)
  • Cancel Reasons: Bankers select a reason before cancelling; reason is stored and shown to the requester
  • Requests Archive: Requests older than a configurable threshold automatically move to an Archive tab, keeping the main view focused
  • Stale Request Management: Officers can auto-tombstone stale open requests and bulk-cancel them via the "Cancel Stale" button
  • Priority System: Mark requests as Rush, High, Normal, or Low priority for better organization
  • Officer Controls: Officers can configure guild-wide request percentage limits (1-100% of inventory)

Automated Order Fulfillment

  • One-Click Fulfill: Bankers click "Fulfill" button—addon automatically attaches items to mail
  • Smart Stack Splitting: Automatically splits oversized stacks with confirmation dialogs
    • Example: Request for 2 Felcloth from a 5-stack? Addon prompts to split precisely 2 items
    • Complex requests (e.g., 175 Runecloth from mixed stacks) handled automatically
  • Optimal Stack Selection: Intelligent bin-packing algorithm selects best stack combinations
  • Visual Indicators: Button icons change to show split needed vs. ready to mail

Blazing-Fast Synchronization

  • Delta Sync Protocol: Only syncs what changed—achieves 90-99% bandwidth reduction
  • Link-Less Optimization: Removes redundant item links for additional bandwidth savings (v0.8.0+)
  • Pull-Based Handshake: Clients request only the data they need, when they need it
  • P2P Collect/Offer Pipeline: Hash syncs run through a coordinated collect/offer/dispatch cycle with broadcast collision prevention—no more corrupt message errors on busy guild channels
  • AceCommQueue-1.0: Embedded send-queue library ensures no two messages on the same prefix ever interleave chunks in transit — fully transparent, no call site changes required
  • Compact Request Protocol: Request index and records sync in a compact positional format — ~60% less bandwidth than the previous key-value format
  • Event-Sourced Requests: Append-only request log prevents conflicts and data loss
  • Offline Resilience: Players who were offline automatically catch up when they log in

How It Works

Setup (One-Time)

  1. Bank Characters: Add "gbank" to character's Public Note or Officer Note
  2. Enable Reporting: In AddOns settings, enable "Report bank contents" and "Scan bank on open"
  3. Scan Inventory: Open your bank/bags/mail—addon scans and shares with guild
  4. Done! Inventory is now visible to all guild members using TOGBankClassic

Daily Usage

For Guild Members:

  1. Open TOGBankClassic (minimap button or /togbank show)
  2. Click "Search" → type item name (e.g., "Felcloth")
  3. Click result → dialog shows available quantity
  4. Enter amount → click "Send Request"
  5. Wait for mail notification—it's that easy!

For Bank Alts:

  1. Open TOGBankClassic → click "Requests"
  2. See all pending requests with quantities
  3. Click "Fulfill" on any request
  4. Addon attaches items (auto-splits if needed)
  5. Type recipient name → send mail → done!

Slash Commands

Basic Commands

  • /togbank or /togbank show - Open main inventory window
  • /togbank sync - Manually trigger full guild sync
  • /togbank share - Broadcast your inventory/requests to guild
  • /togbank wipe - Reset your local database (keeps character data)
  • /togbank wipeall - Reset entire database (fresh start)
  • /togbank wipeframes - Reset window positions to defaults

Debug Commands (Advanced)

  • /togbank debug - Toggle debug output in chat
  • /togbank hashdebug - Show banker hash freshness (diagnose sync issues)
  • /togbank deltastats - Display P2P synchronization telemetry
  • /togbank perfmetrics - Show memory usage and performance stats

Configuration

Access Options

  • Press ESC → Interface → AddOns → TOGBankClassic
  • OR: /togbank options

Key Settings

  • Bank Section: Enable reporting, auto-scan on bank open, mail scan settings
  • Request Section: Configure request limits, priority display, mute warnings
  • UI Section: Minimap button visibility, window opacity
  • Debug Section: Enable persistent logging, toggle debug categories
